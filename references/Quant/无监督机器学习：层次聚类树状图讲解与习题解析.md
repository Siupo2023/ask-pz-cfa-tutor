#### 知识精讲

本课程讲解层次聚类的可视化工具**系统树图（树状图）**，并完整总结无监督学习的两大任务：降维（主成分分析）与聚类（K均值、层次聚类），最后结合真题讲解知识点的应用。

**层次聚类的可视化：系统树图（树状图）**
* **基本作用**：以树状结构直观展示层次聚类的自底向上聚合过程，将逐步分组的结果可视化呈现。
   - 初始状态：11个独立数据点，每个点单独为一类，对应树状图的最底层。
   - 聚合过程：按照数据点/簇之间的相似度逐步合并，从11类合并为6类，再继续合并为最终的2大类，符合自底向上凝聚式层次聚类的逻辑。
* **坐标轴含义**：
   - 横轴：代表不同的聚类簇，单个数据点、合并后的小簇、大簇都对应横轴上的分组。
   - 🚩重点：纵轴代表距离度量，可使用欧氏距离等距离计算方法，是理解树状图的核心。
* **图形元素含义**：
   - 拱形（ARC）：由两条垂直线+一条水平线连接而成，一个拱形代表一个聚类簇（也称为树突）。
   - 🚩重点：垂直线的高度代表被合并的两个簇之间的距离，高度越低说明两个簇相似度越高、距离越近。
* **截断虚线的含义**：水平虚线代表指定的距离阈值，虚线穿过的拱形数量就是该距离阈值下最终得到的聚类数：
   - 当阈值为0.003时，无法合并任何原始点，共得到11个类别；
   - 当阈值为0.035时，可以合并原始点为小簇，但无法继续合并大簇，共得到6个类别；
   - 当阈值为0.048时，可以继续合并得到2个大类别。

**无监督学习整体框架总结**
* **无监督学习核心定义**：使用未标注数据（无目标变量、无输出），核心任务分为两类：降维和聚类。
* **降维算法：主成分分析（PCA）**
   - **应用场景**：当原始数据维度高、特征过多时，提前降维避免模型过拟合，简化模型。
   - **核心目标**：从原始特征中构造少数几个互不相关的新组合特征（主成分），保留原始数据的大部分信息。
   - **核心概念**：
     1. 主成分（PC1/PC2/...）：本质是原始特征的线性组合，相当于对原始坐标轴做旋转变换，也称为**特征向量（eigen vector）**。
     2. 🚩重点：主成分按照**特征值（eigen value）**从大到小排序，特征值衡量对应主成分的方差，方差越大代表该主成分保留的信息量越大，特征值对应该主成分的方差贡献率。
     3. 主成分选择停止规则：当累积方差贡献率达到85%~95%时停止选择，可通过碎石图辅助判断。
     4. 🚩重点：最终选出的所有主成分之间相互垂直、互不相关。
   - **优化目标**：最小化投影误差（所有数据点到新坐标轴的距离和最小），最大化投影后数据的离散程度（方差，即spread）。
   - **缺点与应用**：缺点是结果可解释性弱，类似黑箱；常作为其他算法的预处理步骤，先降维简化数据再建模，避免过拟合。

* **聚类算法核心目标**：将相似数据聚为同一类，最终要求：类内相似度高（内聚）、类间相似度低（分离）。
* **聚类应用（投资领域）**：对上市公司分组，辅助构建投资组合，优化分散化效果。
* **常见聚类算法1：K均值（KMeans）**
   - 🚩重点：**K是需要提前指定的超参数**，属于算法的核心输入。
   - 算法步骤：
     1. 随机指定K个初始聚类中心（质心）；
     2. 计算每个样本到各个质心的距离，将样本分配给距离最近的质心，形成K个簇；
     3. 重新计算每个簇内所有样本的平均值，作为新的质心；
     4. 重复步骤2-3，直到没有样本需要重新分配、质心不再变化，迭代停止。
   - 优缺点：优点是计算速度快、数据量较大时效果好、原理直观易理解；缺点是K需要主观提前指定，聚类结果依赖初始质心的选择。

* **常见聚类算法2：层次聚类**
   - 🚩重点：与K均值的核心区别：不需要提前指定聚类数，会生成所有中间层次的聚类结果，便于分析师从多个层次考察数据的分割，更加客观。
   - 分类：
     1. 自底向上（凝聚法，bottom-up）：先将每个样本单独作为一类，逐步合并相似的簇，适合对数据不熟悉、先探索局部再汇总为大类的场景；
     2. 自顶向下（分裂法，top-down）：先将所有样本作为一类，逐步分裂为更小的簇，适合已经掌握全局特征、再逐步细分的场景。

**无监督学习真题解析**
* **例题1：判断哪项需求需要机器学习聚类算法**
  需求背景：基金经理希望排查基金持股的集中度风险，优化分散化效果，提出三个方案：
  1. 方案1：对比基金持仓敞口与基准敞口的差异：不需要机器学习，用回归分析即可实现；
  2. 方案2：基于8个运营、财务指标对基金持仓股票分组，考察分组是否存在集中性偏差：属于聚类问题，需要用到无监督机器学习的聚类算法；
  3. 方案3：将基金收益率对市场指数和行业指数做回归，考察回归系数：不需要机器学习，普通回归分析即可实现；
  结论：只有方案2需要用机器学习聚类算法实现。

* **例题2：聚类算法所需输入信息判断**
  问题：KMeans聚类算法除了距离计算、超参数K之外，还需要什么信息？
  🚩重点：还需要提前指定**特征集**：计算样本之间的距离，必须明确基于哪些特征计算距离，不是所有特征都会被使用，因此特征集是必须提前给定的必要输入。

[00:00:33](https://getnotes.seek:33) **树状图的聚合过程演示**
示例中共有从A到K共11个原始数据点，初始状态每个数据点单独作为一类；之后按照相似度合并，将AB合并为一类、CD合并为一类、EF合并为一类、G单独为一类、HI合并为一类、JK合并为一类，得到第二层共6个类，符合自底向上凝聚式聚类的逻辑；之后继续合并相似度高的簇，将AB、CD、EF三个小簇合并为一个大类，最终得到7、8两个大簇。

[00:02:10](https://getnotes.seek:130) **树状图坐标轴含义讲解**
树状图的坐标轴都有实际意义：横轴代表聚类簇，单个点或合并后的簇都对应横轴的分组；**纵轴代表距离度量**，这是理解树状图的关键，通常使用欧氏距离计算。

[00:02:47](https://getnotes.seek:167) **树状图图形元素含义讲解**
树状图中的拱形由两条垂直线和一条水平线连接而成，一个拱形代表一个聚类簇，也称为树突，整个图形因此称为树状图。垂直线的高度代表被合并的两个簇之间的距离，例如AB合并的垂直线高度对应AB之间的距离0.02，CD合并的距离约为0.01。

[00:04:29](https://getnotes.seek:269) **水平截断虚线的含义讲解**
树状图中的水平虚线代表指定的距离阈值，其作用是判断在该距离下可以得到多少个聚类：
- 当虚线对应纵轴距离为0.003时，AB之间距离为0.02大于0.003，无法合并，因此所有原始点都单独为一类，共11个类别；
- 当虚线对应距离为0.035时，原始点可以合并为6个小簇，但7号大类内部合并距离为0.042，大于0.035无法合并，因此共得到6个类别；
- 当虚线对应距离为0.048时，可以合并得到两个大类别。
树突高度越低，代表两个簇距离越近、相似度越高。

无监督学习使用未打标签的数据，数据没有目标变量、没有输出，核心任务只有两类：降维和聚类。

[00:08:46](https://getnotes.seek:526) **降维算法主成分分析回顾**
降维算法主要就是主成分分析（PCA），应用场景是原始数据维度高、特征多，直接建模容易导致模型过拟合，因此需要提前降维。核心目标是从原始特征中构造少数几个互不相关的新组合特征，不是直接删除原始特征，而是生成新的线性组合特征。

[00:10:00](https://getnotes.seek:600) **主成分分析核心知识点梳理**
1. 主成分PC1/PC2/PC3...也称为特征向量，本质是原始特征的线性组合，相当于对原始坐标轴做旋转变换。
2. 主成分按照特征值从大到小排序，特征值衡量对应主成分的方差，投影后方差越大说明保留的信息量越多，特征值对应该主成分的方差贡献率，也就是能解释的总信息量比例。特征值最大的就是第一主成分。
3. 主成分选择的两个优化方向：最小化投影误差（数据点到新坐标轴的距离和最小，让新坐标轴尽可能贴合原始数据），最大化投影后数据的离散程度（spread，即方差，也就是特征值）。
4. 选择停止规则：累积方差贡献率达到85%~95%就停止，可通过碎石图辅助判断。
5. 🚩重点：选出的所有主成分之间相互垂直，因此互不相关。

[00:13:29](https://getnotes.seek:809) **主成分分析的缺点与应用**
缺点：结果可解释性差，类似黑箱；应用：通常作为其他算法的预处理步骤，先降维简化数据，避免过拟合，让后续模型更简单。

[00:14:00](https://getnotes.seek:840) **聚类算法核心内容回顾**
聚类的目标是把相似数据聚为同一类，最终要求是类内高内聚、类间高分离。聚类在投资领域的应用包括对公司分组、辅助构建投资组合优化分散化。常见算法有两种：K均值和层次聚类。

[00:14:48](https://getnotes.seek:888) **K均值算法步骤回顾**
K是需要提前指定的超参数，比如指定K=3就是要把数据分为3类。算法步骤为：
1. 随机指定K个初始质心（聚类中心）；
2. 计算每个样本到各个质心的距离，把样本分配给距离最近的质心，形成K个簇；
3. 计算每个簇样本的平均值，作为新的质心；
4. 重复步骤2-3，直到没有样本需要重新分配、质心不再变化，迭代停止，聚类完成。

[00:16:50](https://getnotes.seek:1010) **K均值算法优缺点回顾**
优点：计算速度快，大数据量下效果好，原理直观容易可视化；缺点：K需要提前主观指定，无法客观确定，且聚类结果依赖初始质心的选择，这和层次聚类形成明显区别。

[00:17:22](https://getnotes.seek:1042) **层次聚类核心特点回顾**
🚩重点：层次聚类和K均值最大的区别是不需要提前指定聚类数，会生成所有中间层次的聚类结果，便于分析师考察多个层次的数据分割，比K均值更客观。分为自底向上凝聚和自顶向下分裂两类：自底向上适合不熟悉数据，先探索局部再合并为大类；自顶向下适合已经掌握全局，再逐步细分。系统树图只需要认识名词，不需要重点掌握。
